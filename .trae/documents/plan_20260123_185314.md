# 修复data_loader.py中的AttributeError

## 1. 错误分析
- 错误位置：`data_loader.py` 第198行
- 错误原因：`data_config.model_transforms` 为 `None`，导致访问 `inputs` 属性失败
- 影响范围：`transform_dataset` 和 `transform_iterable_dataset` 函数

## 2. 修复方案
- 修改 `transform_dataset` 函数，添加对 `model_transforms` 为 `None` 的检查
- 修改 `transform_iterable_dataset` 函数，添加相同的检查
- 当 `model_transforms` 为 `None` 时，使用空列表替代 `model_transforms.inputs`

## 3. 具体修改

### 3.1 修改 `transform_dataset` 函数
在 `transform_dataset` 函数中，将：
```python
*data_config.model_transforms.inputs,
```
替换为：
```python
*(data_config.model_transforms.inputs if data_config.model_transforms is not None else []),
```

### 3.2 修改 `transform_iterable_dataset` 函数
在 `transform_iterable_dataset` 函数中，将：
```python
*data_config.model_transforms.inputs,
```
替换为：
```python
*(data_config.model_transforms.inputs if data_config.model_transforms is not None else []),
```

## 4. 预期结果
- 修复后的数据加载器能够处理 `model_transforms` 为 `None` 的情况
- 训练过程能够正常进行，不再出现 AttributeError
- 保持对现有配置的兼容性
